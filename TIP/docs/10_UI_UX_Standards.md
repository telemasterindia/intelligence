# UI UX Standards

TIP should feel like an internal operations tool: clear, dense, calm, and fast.

## Product Feel

- Work-focused rather than marketing-focused.
- Tables, filters, status indicators, and forms should be easy to scan.
- Important operational states should be visible without decorative noise.
- Screens should help users act, not just observe.

## Navigation

Primary areas should align with platform modules:

- Dashboard.
- Imports.
- Vendors.
- Campaigns.
- Leads.
- Intelligence.
- Recommendations.
- Settings.

## Interface Rules

- Use tables for operational lists.
- Use status badges for lifecycle states.
- Use filters for vendor, campaign, date, geography, and severity.
- Use clear empty states that tell the user what action is available.
- Keep business explanations close to the data they explain.
- Avoid hiding audit or validation detail behind vague summaries.

## User Trust

TIP users should always be able to see:

- Where data came from.
- Why a row was rejected.
- Which rule affected a score or recommendation.
- When a record was updated.
- Whether a metric is raw, aggregated, scored, or predicted.
