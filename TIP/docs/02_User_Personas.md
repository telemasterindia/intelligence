# User Personas

TIP is built for a small internal team. The product should feel operational, fast, and practical rather than broad or consumer-facing.

## Owner or Executive

Needs:

- See vendor quality and ROI.
- Understand campaign performance.
- Identify wasted spend.
- Compare business trends over time.
- Trust that reports are based on complete history.

Key questions:

- Are we buying the right data?
- Which vendors should receive more or less budget?
- Which campaigns are profitable?
- Where is revenue being lost?

## Operations Manager

Needs:

- Monitor imports, validation quality, campaign health, and agent outcomes.
- Detect operational bottlenecks.
- Understand why a recommendation was made.
- Review exceptions and rejected rows.

Key questions:

- Which files imported cleanly?
- Which vendors are causing rework?
- Which campaigns need attention today?
- Which lead groups should be called first?

## Data or Admin User

Needs:

- Register vendors and campaigns.
- Import files safely.
- Review validation issues before committing data.
- Maintain settings and business rules.

Key questions:

- Is this file structurally valid?
- What changed between imports?
- Which rows were rejected and why?
- Can this import be audited later?

## Agent or Calling Team Lead

Needs:

- Receive prioritized work queues.
- Understand broad campaign direction.
- Avoid low-quality, risky, or exhausted lead groups.

Key questions:

- What should the team call next?
- Which geography or campaign is performing now?
- Which leads should be avoided or deprioritized?

## Codex or AI Assistant

Needs:

- Clear module boundaries.
- Markdown documentation for product intent and technical contracts.
- Consistent naming.
- Explicit business rules.

Key questions:

- Where should new behavior live?
- Which rule is configurable?
- Which module owns this responsibility?
- What should not be changed?
