# Roadmap

TIP should grow in disciplined layers. Each layer should produce real internal value before the next layer depends on it.

## Sprint 0: Foundation

Completed foundation:

- Repository structure.
- React frontend shell.
- Express backend shell.
- PostgreSQL migration structure.
- Initial documentation.
- Local development workflow.

## Sprint 0.5: Database Refinement

Completed refinement:

- Durable lead identity model.
- Lead occurrence model.
- Outcome history.
- Import validation history.
- Intelligence profile foundation.
- Rule set and rule version foundation.
- Vendor, geography, and calling intelligence tables.

## Sprint 1: Data Ingestion Foundation

Recommended next scope:

- Vendor and campaign basics.
- CSV schema profiling.
- Import batch lifecycle.
- Validation result storage.
- Audit logging service.
- Repository tests.

Avoid in Sprint 1:

- Lead scoring.
- Dashboards.
- AI recommendations.
- ROI analytics.
- Calling logic.

## Later Phases

Potential sequence:

- Lead import commit flow.
- Lead identity matching and duplicate handling.
- Outcome history entry.
- Vendor and campaign dashboards.
- Geography intelligence.
- Rule configuration UI.
- Scoring engines.
- Recommendation engine.
- Predictive intelligence.
- Production hardening and role-based access.
